# เกมคำต้องเชื่อม 
> **credit game:** อ้างอิงกติกาความสนุกมาจากช่อง YouTube: **[ยกกำลัง](https://www.youtube.com/@yokgamlang)**

โปรเจกต์เกมต่อคำภาษาไทยบนเว็บแอปพลิเคชัน พัฒนาด้วย **Streamlit** และระบบประมวลผลภาษาธรรมชาติ **(PyThaiNLP)** รองรับทั้งระบบพิมพ์ด้วยมือและระบบสั่งการด้วยเสียง (Speech Recognition) อัตโนมัติ

## กติกาเกม 
- **การเล่น:** ผู้เล่นแต่ละคนต้องสลับกัน พูดหรือพิมพ์ คำศัพท์ภาษาไทย 1 คำ ที่เชื่อมโยงกับคำก่อนหน้า
- **เงื่อนไขคำศัพท์:** 
    - ต้องเป็นคำนามเท่านั้น
    - ห้ามซ้ำกับคำที่เคยใช้ไปแล้วในเกมนั้น
    - ห้ามมีพยางค์ซ้ำกับพยางค์ใด ๆ ที่เคยถูกบันทึกไว้แล้วเช่นกัน
- **การคัดออก:** ผู้เล่นที่ใช้คำไม่ตรงตามเงื่อนไข (ไม่ใช่คำนาม/คำซ้ำ/พยางค์ซ้ำ) จะถือว่าแพ้ทันที จากนั้นผู้เล่นคนต่อไปจะได้เริ่มคำใหม่ (ไม่จำเป็นต้องเชื่อมกับคำก่อนหน้า)
- **ผู้ชนะ:** เล่นวนไปเรื่อย ๆ จนกว่าจะเหลือผู้เล่นคนสุดท้ายที่รอดชีวิต

## วิธีการติดตั้งและใช้งาน
### 1. เตรียมความพร้อมก่อนติดตั้ง
ระบบรองรับการรับเสียงจากไมโครโฟนโดยตรง หากใช้ Windows/macOS ต้องตรวจสอบสิทธิ์การเข้าถึงไมโครโฟนของตัวเครื่องก่อนใช้งาน

### 2. การติดตั้ง Dependency
ติดตั้งแพ็กเกจหลักผ่าน `uv` หรือ `pip`:
```bash
uv add streamlit pythainlp SpeechRecognition pyaudio
# หรือหากใช้ pip
pip install streamlit pythainlp SpeechRecognition pyaudio
```
(หมายเหตุ: สำหรับผู้ใช้ Windows หากติดตั้ง pyaudio ไม่ผ่าน ให้ดาวน์โหลดไฟล์ .whl ที่ตรงกับเวอร์ชัน Python ของคุณมาติดตั้งแยก)

### 3. วิธีการรันแอปพลิเคชัน
เปิด Terminal ในโฟลเดอร์โปรเจกต์แล้วพิมพ์คำสั่ง
```bash
streamlit run app.py
``` 
ระบบจะเปิดหน้าเว็บเบราว์เซอร์ขึ้นมาที่ http://localhost:8501 เพื่อเริ่มเกม

### 4. คู่มือการใช้งานแอปพลิเคชัน (UI Guide)
อินเตอร์เฟสของแอปพลิเคชันแบ่งออกเป็น 3 ส่วนหลัก ดังนี้:
- 🕹️ ส่วนควบคุมสถานะเกม (ด้านบนสุด):
    - ปุ่ม Restart (clear database): ใช้สำหรับล้างคลังคำศัพท์ทั้งหมดในฐานข้อมูลเพื่อเริ่มเกมรอบใหม่
    - ปุ่ม Pause / Resume: ใช้สำหรับกดหยุดเกมชั่วคราว (เมื่อกด Pause ช่องกรอกคำศัพท์และปุ่มบันทึกทั้งหมดจะถูกล็อกเพื่อไม่ให้ส่งข้อมูลได้)
- 🎤 ส่วนป้อนคำศัพท์ใหม่ (ตรงกลาง):
    - ปุ่มกดเพื่อพูด: คลิก 1 ครั้งแล้วพูดคำศัพท์ภาษาไทย ระบบจะส่งคำไปประมวลผลให้อัตโนมัติเมื่อพูดจบ
    - ช่องป้อนคำใหม่ (พิมพ์มือ): พิมพ์คำศัพท์แล้วกด Enter ระบบจะประมวลผล และล้างข้อความในช่องพิมพ์ให้อัตโนมัติเพื่อรอรับคำถัดไป
- 📚 ส่วนคลังคำศัพท์ (ด้านล่างสุด):
    - แสดงรายการคำศัพท์ทั้งหมดที่เคยตอบผ่านในรอบนั้น เรียงจากใหม่ไปเก่า
    - ปุ่มบันทึก (แก้ไขคำผิด): สามารถคลิกพิมพ์แก้ไขข้อความในกล่องคำศัพท์ได้โดยตรง (ใช้ในกรณีที่ระบบ ตรวจจับเสียงพูดผิด) เมื่อแก้เสร็จให้กดปุ่ม "บันทึก" ระบบจะหั่นพยางค์และอัปเดตลงฐานข้อมูลให้ใหม่ทันที

## รายละเอียดการพัฒนา
### ฟีเจอร์หลักและเทคโนโลยีที่ใช้
โปรเจกต์นี้ขับเคลื่อนด้วย 4 กลไกหลัก โดยแยกตรรกะเบื้องหลังไว้ที่ db_manager.py และการแสดงผลไว้ที่ app.py:
1. ระบบคัดกรองคำนาม (Noun Filtering): 
    - ใช้ pythainlp.tag.pos_tag ร่วมกับคลังข้อมูลสากล corpus="pud" เพื่อเช็กประเภทคำโดด (NOUN, VERB, ADJ) 
2. ระบบแยกพยางค์ (Syllable Tokenization):
    - ใช้ pythainlp.tokenize.syllable_tokenize ในการหั่นคำศัพท์ออกเป็นพยางค์ย่อยก่อนนำไปตรวจสอบการซ้ำในฐานข้อมูล
3. ระบบจำเสียงพูด (Speech Recognition): 
    - ใช้ไลบรารี SpeechRecognition ร่วมกับ pyaudio และดึง API ของ Google Web Speech (th-TH) มาเปลี่ยนเสียงพูดให้เป็นข้อความภาษาไทย
4. ฐานข้อมูล (Database):
    - ใช้ sqlite3 จัดเก็บข้อมูล เพื่อความรวดเร็วในการ Query เช็กพยางค์ซ้ำ 
    - ใช้ st.session_state ของ Streamlit ในการควบคุม Flow ของเกม (Pause/Resume)

### โครงสร้างฐานข้อมูล (Database Architecture)
ระบบใช้ฐานข้อมูล SQLite (data/word.db) ประกอบด้วย 2 ตารางหลัก:
#### 1. ตาราง `words` (เก็บคำศัพท์หลัก)

| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `id` | INTEGER (PK, Auto Inc) | ไอดีอ้างอิงของคำศัพท์แต่ละคำ |
| `word_text` | TEXT (UNIQUE) | ตัวคำศัพท์เต็มๆ (ห้ามซ้ำในตาราง) |

#### 2. ตาราง `syllables` (เก็บแยกพยางค์)
| ชื่อคอลัมน์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `id` | INTEGER (PK, Auto Inc) | ไอดีอ้างอิงของพยางค์ |
| `word_id` | INTEGER (FK) | ลิงก์เชื่อมโยงกลับไปที่ `id` ของตาราง `words` |
| `syllable_text`| TEXT | ข้อความพยางค์ที่ถูกแยกออกมา (ใช้สำหรับค้นหาพยางค์ซ้ำ) |

### โครงสร้างไฟล์ในโปรเจกต์
```
├── data/
│   └── word.db          # ไฟล์ฐานข้อมูล SQLite (ระบบจะสร้างให้อัตโนมัติ)
├── app.py               # ไฟล์หน้าจอหลัก UI และระบบควบคุม Streamlit
├── db_manager.py        # ระบบควบคุมเกม, NLP กรองคำนาม และการติดต่อฐานข้อมูล
└── README.md            # เอกสารประกอบโปรเจกต์
```
